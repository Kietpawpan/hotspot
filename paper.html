บทคัดย่อ
การบริหารจัดการมลพิษทางอากาศระดับภูมิภาคที่ผ่านมาประสบความท้าทายในด้านความรวดเร็วและข้อจำกัดในการเข้าถึงพื้นที่ปฏิบัติงานของเจ้าหน้าที่ ซึ่งการรายงานพิกัดจุดความร้อนผ่านแอปพลิเคชันสนทนาอาจทำให้ข้อมูลเกิดความคลาดเคลื่อนและไม่ครบถ้วน ส่งผลต่อประสิทธิภาพในการสั่งการของศูนย์สื่อสารการแก้ไขปัญหามลพิษทางอากาศ (ศกพ.) ส่วนภูมิภาค ข้อค้นพบเชิงประจักษ์ที่สำคัญคือการแก้ไขปัญหาอย่างยั่งยืนจำเป็นต้องให้ความสำคัญกับการยกระดับกระบวนการทำงานของภาครัฐควบคู่ไปกับการรณรงค์ลดการเผาไหม้ในภาคประชาชน การศึกษาครั้งนี้จึงมีวัตถุประสงค์เพื่อพัฒนาระบบรายงานสถานการณ์คุณภาพอากาศและจุดความร้อนแบบเบ็ดเสร็จ ณ จุดเดียว เพื่อเพิ่มประสิทธิภาพในการบริหารจัดการข้อมูลและยกระดับธรรมาภิบาลในการปฏิบัติงานของสำนักงานสิ่งแวดล้อมและควบคุมมลพิษที่ 11 (นครราชสีมา) (สคพ.11)
ระบบได้รับการออกแบบด้วยสถาปัตยกรรมไร้เครื่องแม่ข่าย (Serverless Architecture) ผสานการทำงานแบบ Microservices โดยใช้บริการอิเล็กทรอนิกส์ (e-Service) ของกระทรวงทรัพยากรธรรมชาติและสิ่งแวดล้อม (ทส.) เป็นประตูทางเข้าหลัก ส่วนแสดงผลได้รับการพัฒนาเป็นเว็บแอปพลิเคชันให้บริการผ่าน GitHub Repository และใช้ Google Apps Script เป็นตัวกลางในการบูรณาการข้อมูลเปิดจากหลายแหล่ง ได้แก่ ข้อมูลคุณภาพอากาศจากกรมควบคุมมลพิษ ข้อมูลอุตุนิยมวิทยาจากกรมอุตุนิยมวิทยา และดึงข้อมูลพิกัดจุดความร้อนโดยตรงจาก API ของ GISTDA สถาปัตยกรรมนี้มีจุดเด่นด้านความคุ้มค่า ความยืดหยุ่นต่อความขัดข้องของระบบ และเปิดโอกาสให้หน่วยงานอื่นสามารถนำไปศึกษาและประยุกต์ใช้ได้โดยสะดวก
ผลสัมฤทธิ์จากการนำระบบไปใช้งานจริงพบว่า การแสดงผลจุดความร้อนแบบเรียลไทม์ช่วยให้ได้ข้อมูลที่แม่นยำและเป็นปัจจุบัน เทคโนโลยีได้เข้ามาเป็นเครื่องมือสนับสนุนการปฏิบัติงานที่ส่งเสริมความรับผิดชอบและเพิ่มความโปร่งใส (Technology-Enabled Accountability) กระตุ้นให้หน่วยปฏิบัติงานสามารถวางแผนลงพื้นที่ระงับเหตุได้อย่างรวดเร็วและรายงานผลได้อย่างเป็นระบบ นำไปสู่การปรับเปลี่ยนกระบวนทัศน์ในการแก้ไขปัญหาสิ่งแวดล้อมที่มุ่งเน้นการเพิ่มขีดความสามารถในการตอบสนองของภาครัฐ สร้างมาตรฐานใหม่ด้านการบูรณาการข้อมูลเพื่อการตัดสินใจอย่างเป็นรูปธรรม
คำสำคัญ: ฝุ่นละออง PM2.5 จุดความร้อน สถาปัตยกรรมไร้เครื่องแม่ข่าย ข้อมูลเปิด ธรรมาภิบาลข้อมูล

1. สภาพปัญหาและเหตุผลความจำเป็น

การบริหารจัดการและแก้ไขปัญหามลพิษทางอากาศในระดับภูมิภาคจำเป็นต้องอาศัยข้อมูลที่ถูกต้องและทันเหตุการณ์ ก่อนหน้าที่จะมีการพัฒนาระบบนี้ศูนย์สื่อสารการแก้ไขปัญหามลพิษทางอากาศ (ศกพ.) ส่วนภูมิภาค 11 พบข้อจำกัดบางประการในการปฏิบัติงานระดับพื้นที่ นั่นคือการพึ่งพาระบบการแจ้งพิกัดจุดความร้อนและรายงานผลผ่านแอปพลิเคชันสนทนา ซึ่งอาจส่งผลให้ข้อมูลมีความคลาดเคลื่อนและไม่ครบถ้วนในบางกรณี

ความท้าทายหลักที่พบคือข้อจำกัดด้านความรวดเร็วในการประสานงานและการลงพื้นที่ โดยในบางเหตุการณ์การรายงานผลการปฏิบัติงานเกิดขึ้นภายหลังจากเหตุการณ์ไฟป่าหรือการเผาไหม้ได้ยุติลงแล้ว นอกจากนี้ยังมีอุปสรรคในการเข้าถึงพื้นที่ห่างไกลทำให้ไม่สามารถตรวจสอบจุดความร้อนได้อย่างครบถ้วนทุกจุด ตลอดจนปัญหาข้อมูลจุดความร้อนเข้าถึงได้ แต่ขาดความสะดวกในการค้นหาและนำไปใช้ในการกำกับดูแลการแก้ไขปัญหามลพิษในพื้นที่รับผิดชอบ

ข้อค้นพบเชิงประจักษ์จากสถานการณ์ดังกล่าวชี้ให้เห็นว่า กลไกการแก้ปัญหาของภาครัฐไม่ควรจำกัดอยู่เพียงการเฝ้าระวังพฤติกรรมการเผาของประชาชนเท่านั้น แต่ควรให้ความสำคัญกับการพัฒนากระบวนการปฏิบัติงานภายในระบบราชการให้มีความรวดเร็ว แม่นยำ และมีประสิทธิภาพมากยิ่งขึ้น การนำเทคโนโลยีสารสนเทศมาช่วยสนับสนุนการตรวจสอบข้อมูลเชิงลึกข้ามระบบจึงเป็นสิ่งจำเป็นอย่างยิ่ง

เพื่อแก้ไขข้อจำกัดดังกล่าวและยกระดับธรรมาภิบาลในการทำงานจึงได้พัฒนาระบบรายงานสถานการณ์คุณภาพอากาศแบบศูนย์รวมสำหรับพื้นที่รับผิดชอบของ สคพ.11 (นครราชสีมา ชัยภูมิ บุรีรัมย์ สุรินทร์) โดยเปลี่ยนรูปแบบจากการรายงานข้อมูลผ่านบุคคลมาเป็นการเชื่อมโยงข้อมูลพิกัดจุดความร้อนและเวลาที่ตรวจพบโดยตรงจากฐานข้อมูลของ GISTDA เพื่อแสดงผลแบบเรียลไทม์ การนำเทคโนโลยีมาเป็นกลไกสนับสนุนการทำงานทำให้ได้ข้อมูลเชิงประจักษ์ที่โปร่งใสและตรวจสอบได้ ช่วยให้เจ้าหน้าที่สามารถวางแผนลงพื้นที่แก้ไขปัญหาได้อย่างรวดเร็วและตรงจุดมากยิ่งขึ้น

2. สถาปัตยกรรมของระบบ (System Architecture)

ระบบได้รับการออกแบบภายใต้แนวคิด Serverless Architecture และ Microservices โดยลดการพึ่งพาเครื่องแม่ข่ายส่วนบุคคลแต่ใช้การบูรณาการบริการคลาวด์สาธารณะเข้าด้วยกัน แบ่งโครงสร้างการทำงานออกเป็น 3 ส่วนหลัก ดังนี้

ภาพที่ 1. โครงสร้างสถาปัตยกรรมระบบรายงานสถานการณ์คุณภาพอากาศแบบบูรณาการ

2.1. ส่วนเชื่อมโยงผู้ใช้งาน (User Gateway & Routing Layer)

ระบบใช้กลไกการเปลี่ยนเส้นทางผ่านบริการอิเล็กทรอนิกส์ของ ทส. โดยกำหนดชื่อโดเมนทางการเพื่อเป็นประตูทางเข้าหลัก วิธีการนี้ช่วยสร้างความน่าเชื่อถือให้กับระบบและจะทำการส่งต่อผู้ใช้งานไปยังหน้าแดชบอร์ดโดยอัตโนมัติ

2.2. ส่วนแสดงผลและเชื่อมต่อผู้ใช้ (Presentation Layer)

พัฒนาในรูปแบบ Web Application แบบหน้าเดียวโดยใช้เทคโนโลยี HTML5 CSS3 และ JavaScript ระบบส่วนหน้าถูกนำไปจัดเก็บเพื่อให้บริการบน GitHub Repository ซึ่งเป็นบริการเว็บโฮสติ้งที่มีความเสถียรสูงสามารถรองรับปริมาณการเข้าใช้งานจำนวนมากได้อย่างมีประสิทธิภาพ การประมวลผลและการจัดรูปแบบการแสดงผลทั้งหมดดำเนินการที่ฝั่งเครื่องของผู้ใช้งาน

2.3. ส่วนบูรณาการข้อมูลและประมวลผล (Data Integration & Middleware Layer)

ระบบใช้วิธีการเชื่อมโยงข้อมูลจากแหล่งข้อมูลเปิดต่างระบบมารวมไว้ในหน้าจอเดียว ได้แก่

  • ข้อมูลคุณภาพอากาศประจำวันใช้วิธี Static Hyperlinking เชื่อมโยงไฟล์เอกสารรายงานจากระบบ Air4Thai
  • ข้อมูลอัตราการระบายอากาศใช้วิธี Image Embedding ดึงภาพกราฟิกจากเครื่องแม่ข่ายของกรมอุตุนิยมวิทยา
  • ข้อมูลสภาพอากาศแบบเรียลไทม์พัฒนา API Proxy ผ่าน Google Apps Script เพื่อดึงข้อมูลจากฐานข้อมูล JSON ของกรมอุตุนิยมวิทยาช่วยแก้ไขข้อจำกัดด้านความปลอดภัยข้ามโดเมน
  • ข้อมูลสถิติ PM2.5 และจุดความร้อนย้อนหลังประยุกต์ใช้ Google Sheets เป็นระบบฐานข้อมูลโดยใช้ Google Apps Script พัฒนาเป็น RESTful API เพื่อส่งออกข้อมูลในรูปแบบ JSON
  • ข้อมูลพิกัดจุดความร้อนใช้การเรียก API โดยตรงจากฐานข้อมูลของ GISTDA เพื่อสร้างแผนที่เชิงโต้ตอบที่ให้ข้อมูลเวลาและพิกัดที่แม่นยำ

3. จุดเด่นและข้อได้เปรียบทางวิศวกรรมซอฟต์แวร์

3.1. ความคุ้มค่าและโปร่งใส (Cost-Effective & Transparent)

ไม่มีค่าใช้จ่ายในการบำรุงรักษาเครื่องแม่ข่ายและช่วยลดขั้นตอนการรายงานข้อมูลทับซ้อนทำให้ผู้บริหารได้รับข้อมูลสถานการณ์จริงในพื้นที่อย่างรวดเร็ว

3.2. ความสามารถในการทำวิศวกรรมย้อนกลับ (Reverse Engineerability)

โครงสร้างรหัสคำสั่งแบบแยกส่วนเอื้อให้หน่วยงานอื่นสามารถนำซอร์สโค้ดไปศึกษาและปรับใช้เพื่อสร้างระบบของตนเองได้อย่างสะดวก

3.3. ความยืดหยุ่นต่อความขัดข้อง (Fault Tolerance)

การเชื่อมโยงข้อมูลทำงานแยกส่วนกันแบบอิสระ หากแหล่งข้อมูลใดเกิดความขัดข้องระบบจะยังคงแสดงผลข้อมูลส่วนอื่นได้อย่างเป็นปกติ

4. ผลสัมฤทธิ์ที่ได้จากการนำระบบไปใช้งานจริง (Outcomes & Impact)

การนำระบบแดชบอร์ดแบบบูรณาการข้อมูลเปิดมาใช้บริหารจัดการมลพิษทางอากาศในพื้นที่รับผิดชอบของ สคพ.11 ก่อให้เกิดผลสัมฤทธิ์ที่สำคัญทั้งในมิติของการเพิ่มประสิทธิภาพการบริหารจัดการและมิติของการยกระดับธรรมาภิบาลในการปฏิบัติงานของภาครัฐ โดยมีรายละเอียดดังนี้

4.1. การลดขั้นตอนและเพิ่มความแม่นยำของข้อมูล (Streamlining Data Accuracy)

การเชื่อมต่อข้อมูลผ่าน API ของ GISTDA โดยตรงช่วยลดขั้นตอนการรายงาน ระบบสามารถแสดงพิกัดและเวลาการเกิดจุดความร้อนตามความเป็นจริงในทันที ช่วยแก้ปัญหาความคลาดเคลื่อนหรือความล่าช้าในการรายงานผล ส่งผลให้ฐานข้อมูลที่ใช้ในการตัดสินใจมีความถูกต้องและโปร่งใสสูงสุด

4.2. การใช้เทคโนโลยีเพื่อสนับสนุนและยกระดับประสิทธิภาพการทำงาน (Technology-Enabled Performance)

เมื่อข้อมูลเชิงพื้นที่ถูกแสดงผลบนหน้าจอส่วนกลางตลอด 24. ชั่วโมง ผู้บริหารและ ศกพ. ส่วนภูมิภาคสามารถติดตามและประเมินสถานการณ์ได้อย่างแม่นยำ กลไกทางเทคโนโลยีนี้ทำหน้าที่เป็นเครื่องมือสนับสนุนการปฏิบัติงานที่ช่วยให้เจ้าหน้าที่สามารถวางแผนลงพื้นที่ระงับเหตุได้อย่างรวดเร็วและมีประสิทธิภาพ เป็นการเสริมสร้างความเข้มแข็งในระบบการทำงานของราชการได้อย่างเป็นรูปธรรม

4.3. การเสริมสร้างประสิทธิภาพในการสั่งการและประสานงาน (Efficiency in Command and Coordination)

ผู้บริหารสามารถประเมินข้อมูลรอบด้านทั้งค่าฝุ่นละออง ทิศทางลม อัตราการระบายอากาศ และพิกัดจุดความร้อนได้จากจุดเดียว ช่วยลดระยะเวลาในการวิเคราะห์สถานการณ์และทำให้การสั่งการไปยังหน่วยปฏิบัติมีความแม่นยำ สามารถประสานงานเพื่อเข้าควบคุมแหล่งกำเนิดมลพิษได้ทันท่วงทีก่อนที่สถานการณ์จะลุกลาม

4.4. การยกระดับกระบวนทัศน์ในการบริหารจัดการ (Paradigm Shift in Public Sector Operations)

ผลสัมฤทธิ์ที่สำคัญของการนำระบบนี้มาใช้คือการขยายมุมมองในการแก้ไขปัญหา โดยตระหนักว่าความสำเร็จในการลดฝุ่นควันและจุดความร้อนนั้นขึ้นอยู่กับความพร้อมและศักยภาพในการตอบสนองของหน่วยงานภาครัฐเป็นสำคัญ การนำเทคโนโลยีมาช่วยเพิ่มประสิทธิภาพภายในองค์กรจึงเป็นก้าวสำคัญของการพัฒนาระบบการทำงาน เพื่อให้การบริการสาธารณะด้านสิ่งแวดล้อมเกิดประโยชน์สูงสุดต่อประชาชน

5. บทสรุป

การพัฒนาระบบรายงานสถานการณ์ฝุ่นละออง PM2.5 ของ สคพ.11 เป็นการสร้างเครื่องมือที่มีประสิทธิภาพในการสนับสนุนภารกิจของ ศกพ. ส่วนภูมิภาค และสะท้อนถึงแนวทางการบริหารจัดการภาครัฐยุคใหม่ที่มุ่งมั่นพัฒนาขีดความสามารถขององค์กร การบูรณาการข้อมูลเปิดด้วยเครื่องมือที่เหมาะสมผนวกกับวิศวกรรมซอฟต์แวร์ที่ยืดหยุ่นได้สร้างระบบการทำงานที่โปร่งใส สนับสนุนความรับผิดชอบในการปฏิบัติงาน และยกระดับการจัดการทรัพยากรธรรมชาติและสิ่งแวดล้อมให้มีความเข้มแข็ง ซึ่งพร้อมเป็นต้นแบบสำหรับการนำไปประยุกต์ใช้ในหน่วยงานอื่นต่อไป
