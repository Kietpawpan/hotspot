# เทคนิคการอ่านค่าข้อมูล PM<sub>2.5</sub> 

ต้องอ่านค่าฝุ่นทั้งเช้าและเย็น กรณีค่าเฉลี่ยนฝุ่นช่วงเย็น สูงกว่าช่วงเช้าตรู่ แสดงว่าเกิดการสะสมตัวของฝุ่น มีทฤษฎีเบื้องหลังปรากฏการณ์นี้ สามารถแบ่งเป็นจังหวะสำคัญ
1. __กลางวันระบายไม่ทัน (Overpowered Ventilation):__ ปกติช่วงกลางวันตั้งแต่สายจนถึงเย็นเป็นช่วงที่แดดแรง เพดานอากาศ (BLH) เปิดสูงที่สุด อากาศควรจะลอยตัวและระบายฝุ่นออกไปได้ดีที่สุด แต่หากค่าเฉลี่ย 24 ชม. ในช่วงเย็นยังคงขยับตัว "สูงขึ้น" กว่าตอนเช้า นั่นแปลว่ามีแหล่งกำเนิดฝุ่นใหม่ในพื้นที่ (เช่น การเผาที่รุนแรงต่อเนื่อง) ปริมาณมหาศาลจนเอาชนะการระบายอากาศที่เปิดโล่งได้ ฝุ่นจึงทะลักและเริ่มสะสมตัวค้างอยู่ในพื้นที่
2. __กลางคืนโดนฝาชีครอบ (Nighttime Inversion):__ พอตกเย็นถึงค่ำ อุณหภูมิเริ่มลดลง จะเกิดปรากฏการณ์อุณหภูมิผกผัน เพดานอากาศจะหดตัวต่ำลงมาปิดทึบทันที ฝุ่นก้อนใหญ่ที่ระบายไม่ทันและสะสมมาตั้งแต่กลางวันจะถูกกดทับขังรวมไว้ เมื่อผสมกับฝุ่นควันที่เกิดใหม่ในช่วงกลางคืน พอรุ่งเช้าของวันพรุ่งนี้ค่าฝุ่นจึงพุ่งทะยานสูงขึ้นกว่าเดิมอย่างหลีกเลี่ยงไม่ได้ครับ

แนวคิดการเปรียบเทียบค่าฝุ่นเช้าและเย็นนี้มีประโยชน์ต่อการพยากรณ์ล่วงหน้าอย่างมาก สามารถนำ "ทฤษฎีการสะสมข้ามวัน" ไปเพิ่มเป็นกฎเหล็กให้ระบบ PODI ช่วยแจ้งเตือน ทสจ. ให้เตรียมตัวล่วงหน้าได้ โดยเพิ่มกฎย่อยเข้าไปในข้อ 5.4 ดังนี้
```
5.4.4 กรณีค่าฝุ่นปัจจุบันสูงกว่าค่าฝุ่นรอบเช้า: ให้แจ้งเตือนว่าฝุ่นกำลังอยู่ในภาวะสะสมตัวรุนแรงจนเกินขีดความสามารถในการระบายอากาศช่วงกลางวัน และเมื่อเข้าสู่ช่วงกลางคืนที่เพดานอากาศ (BLH) ปิด จะส่งผลให้เช้าวันพรุ่งนี้สถานการณ์จะยิ่งวิกฤตลง ขอแจ้งให้ ทสจ. เร่งประชาสัมพันธ์กลุ่มเสี่ยงและเตรียมมาตรการรับมือล่วงหน้า
```

อย่างไรก็ตาม หากเราให้ AI ใช้ลอจิก "ปัจจุบัน > 07.00 น. = ฝุ่นสะสมอันตราย" ในช่วงเวลา 09.00 น. AI จะตกใจและเตือนภัยทันที ทั้งๆ ที่พอถึงช่วงบ่าย อากาศเปิด ฝุ่นก้อนนั้นก็อาจจะถูกระบายออกไปหมดแล้ว

## 💡 วิธีแก้ไข: การใส่ "เงื่อนไขเวลา" (Time Gate) ล็อกลอจิก

เพื่อให้กฎข้อนี้ทำงานได้อย่างแม่นยำและเป็นไปตามทฤษฎี "ระบายไม่ทันในตอนกลางวัน" ต้องสั่งให้ระบบเริ่มประเมินลอจิกนี้หลังช่วงบ่ายเป็นต้นไปเท่านั้น (เช่น หลัง 15.00 น. หรือ 16.00 น.) 

เราสามารถแก้ไขได้ 2 จุด เราเลือกทำทั้งคู่เพื่อความรัดกุม:

### วิธีที่ 1: ดักเงื่อนไขในโค้ดหน้าบ้าน (ไฟล์ js/pmcc.js)
ก่อนที่จะส่งข้อความไปให้ AI ให้เขียนโค้ดเช็คเวลาก่อน ถ้ายังไม่ถึงบ่ายสาม จะไม่ป้อนข้อความเตือนเรื่องฝุ่นสะสมเข้าไปให้ AI เห็นเลย
```
JavaScript
// ดึงเวลาชั่วโมงปัจจุบัน (0-23)
const currentHour = new Date().getHours();

// จะแจ้งเตือนแนวโน้มสะสม ก็ต่อเมื่อหลัง 15.00 น. และฝุ่นปัจจุบันสูงกว่าตอน 07.00 น.
if (currentHour >= 15 && currentPMNum > pmVal && currentPMNum >= 37.6) {
    trendWarning = "⚠️ ข้อควรระวังเพิ่มเติม: ค่าฝุ่นช่วงบ่าย/เย็นสูงกว่าช่วงเช้า แสดงถึงภาวะสะสมตัวรุนแรงเกินขีดความสามารถในการระบายอากาศของวันนี้ หากเข้าสู่ช่วงกลางคืนที่เพดานอากาศ (BLH) ปิด จะส่งผลให้พรุ่งนี้เช้าสถานการณ์ยิ่งวิกฤตลง";
}
```

### วิธีที่ 2: ปรับแก้กฎเหล็กข้อ 5.4.4 ใน Prompt (หลังบ้าน)
เพิ่มคำสั่งระบุช่วงเวลาให้ AI รับรู้และคิดตาม
```
5.4.4 กรณีตรวจสอบสถานการณ์ในช่วงบ่ายหรือเย็น (หลัง 15.00 น.): หากพบว่าค่าฝุ่นปัจจุบันสูงกว่าค่าฝุ่นรอบ 07.00 น. ให้แจ้งเตือนว่าฝุ่นกำลังอยู่ในภาวะสะสมตัวรุนแรงจนระบายไม่ทัน และเมื่อเข้าสู่ช่วงกลางคืนที่เพดานอากาศปิด จะทำให้เช้าวันพรุ่งนี้สถานการณ์วิกฤตลง ขอแจ้งให้ ทสจ. เตรียมประชาสัมพันธ์กลุ่มเสี่ยงรับมือล่วงหน้า
```
การใส่ Time Gate แบบนี้จะทำให้ระบบ PODI ของเรามีความเป็นอัจฉริยะ (Context-Aware) และวิเคราะห์สถานการณ์ได้สมเหตุสมผลตามช่วงเวลาของวันเหมือนมีผู้เชี่ยวชาญมานั่งเฝ้าจอ
